# Domain Rules

Last reviewed: 2026-06-04

## Financial Documents

- The four primary document types are tax invoice, receipt, cash bill, and delivery note.
- Document numbers follow per-type prefixes: `INV`, `RCP`, `CSB`, `DLV`.
- Document number format is `{PREFIX}-{YYYYMM}-{NNNN}`.
- Document numbers must be unique per tenant and document type.
- Document history must preserve issued financial records; cancellation should be explicit and auditable.

## VAT and Money

- Thai VAT default rate is 7%.
- Backend monetary storage uses `decimal(18,4)`.
- Frontend display should use Thai locale formatting and normally show 2 decimal places for money.
- VAT may be exclusive or inclusive; frontend and backend formulas must match.
- Any change to subtotal, discount, VAT, or grand total requires Ledger and Argus review.

## Tenant Isolation

- Tenant-owned data must include and enforce `TenantId`.
- Users must not read, update, delete, export, print, or pay documents from another tenant.
- JWT claims should include tenant context and role context.
- Cross-tenant behavior must be tested for APIs that list or fetch tenant-owned records.

## Payment

- Online payment is modeled through `PaymentTransaction` and external gateway references.
- Webhook handling must be idempotent.
- Payment status changes must not trust client-side state alone.
- Payment events may update UI in real time via SignalR, but database state remains authoritative.

## UX and Language

- UI is Thai-first.
- App UI should use Lucide icons instead of emoji.
- Preserve the `Warm Horizon` and `Deep Ocean` themes.
- Billing workflows should minimize cognitive load for accounting users and mobile operators.

## Security

- Do not commit production secrets.
- Uploaded files and generated files must be scoped to the correct tenant and document.
- Webhooks should validate source authenticity where gateway support exists.
- Auth and role checks must be explicit for sensitive operations such as settings, payment, export, upload, and deletion/cancellation.
