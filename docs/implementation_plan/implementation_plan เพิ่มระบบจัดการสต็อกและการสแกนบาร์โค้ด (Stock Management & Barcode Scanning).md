# เพิ่มระบบจัดการสต็อกและการสแกนบาร์โค้ด (Stock Management & Barcode Scanning)

เพิ่มความสามารถในการจัดการสต็อกสินค้าแบบครบวงจร ตั้งแต่การระบุจำนวนคงเหลือในคลัง, การตัดสต็อกอัตโนมัติเมื่อออกบิล, ไปจนถึงการใช้เครื่องสแกนบาร์โค้ดเพื่อเพิ่มรายการสินค้าลงในบิลอย่างรวดเร็ว

> [!WARNING] 
> **User Review Required**
> กรุณาตรวจสอบและอนุมัติแนวทางการทำงาน (Workflow) ของการตัดสต็อก และการทำงานของระบบบาร์โค้ดด้านล่างนี้ ว่าตรงกับความต้องการทางธุรกิจของคุณหรือไม่

## Open Questions
> [!IMPORTANT]
> 1. **การตัดสต็อก (Stock Deduction):** คุณต้องการให้ระบบตัดสต็อก "ทันทีที่กดบันทึกเอกสาร" ประเภทใดบ้าง? (ปกติจะตัดสต็อกเมื่อเปิด บิลเงินสด, ใบส่งของ หรือ ใบกำกับภาษีเต็มรูป แต่จะไม่ตัดสต็อกเมื่อเป็น ใบเสนอราคา หรือ ใบเสร็จรับเงิน)
> 2. **บาร์โค้ด (Barcode):** ระบบสามารถใช้ช่อง **รหัสสินค้า (SKU)** เดิมที่มีอยู่ ให้ทำหน้าที่เป็นบาร์โค้ดด้วยได้เลยหรือไม่? (เพื่อความง่าย ไม่ต้องเพิ่มฟิลด์ใหม่ แค่เปลี่ยนชื่อเป็น รหัสสินค้า / บาร์โค้ด)
> 3. **สแกนบาร์โค้ดในหน้าออกบิล:** การทำงานที่คาดหวังคือ มีช่อง "สแกนบาร์โค้ด" เมื่อสแกนปุ๊บ (แล้วเครื่องยิงบาร์โค้ดส่งคำสั่ง Enter) ระบบจะหาสินค้าชิ้นนั้นแล้วเพิ่มลงในตารางรายการทันที (ถ้ามีอยู่แล้วให้ +1) ถูกต้องไหมครับ?

## Proposed Changes

### Database & Backend
อัปเดต Backend ให้รองรับการปรับปรุงสต็อก (StockQuantity มีอยู่ใน Entity แล้ว แต่ยังไม่ได้เปิดให้ใช้งานผ่าน UI และไม่ได้เขียนลอจิกตัดสต็อก)

#### [MODIFY] `backend/src/SenicBilling.Application/DTOs/Dtos.cs`
- ตรวจสอบให้แน่ใจว่า `StockQuantity` ถูก Map กลับไปกลับมาอย่างถูกต้องระหว่าง Client และ Database

#### [MODIFY] `backend/src/SenicBilling.API/Controllers/DocumentControllerBase.cs`
- เพิ่มลอจิกในการตัดสต็อก (Stock Deduction): ตรวจสอบประเภทเอกสาร หากเป็น `CashBill`, `DeliveryNote` หรือ `TaxInvoice` ระบบจะทำการลดจำนวน `StockQuantity` ในตาราง `Products` ตาม `Quantity` ที่ระบุไว้ในบรรทัดเอกสาร
- คืนสต็อก (Restock): หากมีการลบเอกสารหรืออัปเดตจำนวนสินค้า ระบบจะคำนวณคืนสต็อกให้ถูกต้อง

### Frontend: Product Management
เพิ่มช่องกรอก "จำนวนคงเหลือ (Stock Quantity)" เพื่อให้ผู้ใช้งานสามารถเช็คและตั้งค่าสต็อกตั้งต้นได้

#### [MODIFY] `frontend/src/components/products/ProductsPage.tsx`
- เพิ่มช่อง Input สำหรับ **จำนวนคงเหลือ (Stock Quantity)** ในหน้าต่าง (Modal) เพิ่ม/แก้ไขสินค้า
- แสดงคอลัมน์ **จำนวนคงเหลือ** ในตารางรายการสินค้า เพื่อให้เห็นชัดเจนว่ามีของในสต็อกเท่าไหร่
- เปลี่ยน Label ของ SKU ให้ครอบคลุมถึงบาร์โค้ด (เช่น "รหัสสินค้า / บาร์โค้ด (SKU)")

---

### Frontend: Document/Invoice Form
เพิ่มระบบสแกนบาร์โค้ด และปรับปรุงตารางรายการสินค้า (Line Items)

#### [MODIFY] `frontend/src/components/forms/InvoiceForm.tsx`
- **ระบบสแกนบาร์โค้ด (Barcode Scanner Input):** เพิ่มช่อง `input` เหนือตารางรายการสินค้า เพื่อรอรับค่าจากเครื่องสแกนบาร์โค้ด
- สร้าง Event Listener: เมื่อสแกนบาร์โค้ด ระบบจะค้นหา `Product` ในฐานข้อมูลด้วย SKU
  - หากพบสินค้า: จะเพิ่มรายการนั้นลงในบิล (ถ้ามีในบิลแล้ว ให้บวกจำนวน (Quantity) เพิ่ม 1)
  - หากไม่พบสินค้า: แสดง Error/Warning แจ้งเตือนสั้นๆ
- เปลี่ยนช่อง "รหัส/รายละเอียด" (Description) ให้รองรับการเลือกสินค้าจากฐานข้อมูล (Product Select) นอกเหนือจากการพิมพ์เองแบบอิสระ

6. ทดลองแก้ไขบิลเงินสด เปลี่ยนจำนวนสินค้า แล้วตรวจสอบว่าระบบคำนวณสต็อกชดเชยถูกต้อง
