# **พิมพ์เขียวสถาปัตยกรรมระบบ (System Blueprint)**

## **1\. High-Level Architecture (Cloud-Native SPA/PWA)**

ระบบ Senic Billing Next ใช้สถาปัตยกรรมแบบ **Client-Server Separated (Decoupled)** โดยแบ่งออกเป็น 3 Tiers หลักดังนี้:

### **Tier 1: Presentation Layer (Client-Side)**

* **Platform:** Progressive Web Application (PWA) เข้าถึงผ่าน Browser และติดตั้งเป็นแอปบน Desktop/Mobile ได้  
* **Core Technology:** React 19, TypeScript, Vite  
* **Styling:** Tailwind CSS (ควบคุมโครงสร้าง) \+ Custom CSS Variables (ควบคุม Dynamic Themes)  
* **State Management:** React Context API หรือ Zustand สำหรับจัดการ Global State (เช่น User Session, Theme, Cart/Bill Items)

### **Tier 2: Application / Business Logic Layer (Server-Side API)**

* **Framework:** ASP.NET Core Web API (C\# .NET 10\)  
* **Architecture Pattern:** Clean Architecture / CQRS (Command Query Responsibility Segregation) สำหรับแยกระบบอ่าน (Dashboard) และเขียน (ออกบิล) เพื่อประสิทธิภาพ  
* **ORM:** Entity Framework (EF) Core 10  
* **Security & Auth:** JSON Web Tokens (JWT) สำหรับ Stateless Authentication, Role-based Access Control (RBAC)

### **Tier 3: Data Layer**

* **Database Engine:** PostgreSQL (Relational Database) เหมาะสำหรับข้อมูลธุรกรรมทางการเงินที่มีโครงสร้างชัดเจน และรองรับ JSONB สำหรับข้อมูลที่ยืดหยุ่น  
* **Storage:** Cloud Object Storage (เช่น AWS S3 หรือ Azure Blob) สำหรับเก็บไฟล์โลโก้บริษัท ลายเซ็นอิเล็กทรอนิกส์ และไฟล์ PDF ที่สร้างสำเร็จแล้ว
